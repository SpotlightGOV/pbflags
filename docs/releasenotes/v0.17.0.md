# pbflags v0.17.0

This release introduces **Launches v1** - a new feature rollout system with per-condition value overrides and evaluation scopes, along with significant improvements to the CLI and admin interface.

## Breaking Changes

- Launch configuration format has changed to support inline evaluation and per-condition overrides.
- Code generation now produces per-scope `*Features` types instead of a single unified type.
- Cache keys have been restructured to accommodate the new launch system.

## Migration Steps

- Update launch configurations to use the new v1 format with evaluation scopes and inline conditions.
- Restructure feature flag configurations to use the new `features/` layout with `launches/` subdirectory.
- Update code that references generated feature flag types to use the new per-scope interfaces.
- Run database migrations to support the new launch system schema.

## Server

- **New Launches v1 system** enables sophisticated feature rollouts with per-condition value overrides and evaluation scopes.
- Launch validation is now unified across compile, sync, and validation operations.
- Stale launches not present in compiled bundles are automatically abandoned.
- Database fetcher supports the new launch evaluation model with improved cache handling.

## Code Generation

- Generated code now includes per-scope `*Features` types with duck-typed interfaces for better type safety.
- Java codegen now supports percentage-based rollouts and enhanced feature flag handling.

## Admin Interface

- Launch overrides are now displayed in the conditions table for better visibility.
- Added kill/unkill controls for managing launch states directly from the UI.
- YAML condition comments are now displayed in the admin condition table.
- Launch ramp percentages are shown in condition comments for easier monitoring.

## CLI & Tooling

- **New unified pbflags CLI** replaces separate tools and supports compiled config bundles.
- Enhanced linting includes scope stability checks, condition-scope compatibility validation, and breaking change detection.
- Improved configuration validation with better error reporting and relaxed constraints where appropriate.

## Infrastructure

- Development configuration restructured to use `features/` layout with `launches/` subdirectory.
- Enhanced demo environment with static-value launch overrides and launch lifecycle examples.
- Improved audit logging with proper handling of nil values.
