# pbflags v0.18.0

This release introduces a unified CLI experience and transitions storage from JSON to Protocol Buffers for improved performance, type safety, and easier data model changes in the future.

## Breaking Changes

- Flag conditions are now stored as Protocol Buffers instead of JSON, requiring database migration.
- The database conditions column has changed from JSONB to bytea format.
- *This is a lossy change.* It is only being done this way because we don't believe we
  have any users with persisted conditions.

## CLI

- Added unified `pb` CLI tool that consolidates all pbflags operations.
- Added `pb init` command to initialize new pbflags projects with configuration scaffolding.
- Added `pb migrate` command for database schema migrations.
- Added `pb lint` command to validate flag configurations and launch dimension rules.
- Added `pb format` command to standardize configuration file formatting.
- Added launch lifecycle commands: `pb launch soak`, `pb launch abandon`, and `pb launch land`.
- Added admin commands for flag management: list, get, kill, unkill, and audit operations.
- Flag-specific commands are now grouped under `pb flag` subcommands.

## Server

- Implemented pluggable authentication middleware for the admin API.
- Added secure credential storage for CLI authentication.
- Launch ramps now support config-as-code as authoritative source when `ramp_percentage` is present.
- Enhanced launch dimension validation to ensure dimensions are present in all affected scopes.

## Configuration

- Added `descriptors_path` and `proto_path` options to `.pbflags.yaml` configuration.
- Improved configuration file round-trip marshaling for consistent formatting.

## Migration Steps

- Run `pb migrate` to update your database schema for the new Protocol Buffer storage format.
- Update any existing `.pbflags.yaml` files to include the new `descriptors_path` and `proto_path` fields if needed.
- Replace usage of legacy CLI tools with the new unified `pb` command.
