# pbflags v0.20.0

This release introduces runtime condition overrides with sync locking capabilities and significantly improves the admin UI experience.

## New Features

### Runtime Overrides & Sync Locking
- Added ability to override flag condition values at runtime for testing and emergency fixes
- Introduced sync locking mechanism to prevent configuration conflicts during override operations
- Added comprehensive admin UI for managing condition overrides with age and actor filtering
- Created global overrides listing page for monitoring active runtime changes

### Enhanced Admin UI
- Added new top-level launches index page for better launch management overview
- Introduced expandable rows in launches panel to reveal impacted flags
- Added inline ramp percentage editor directly in the flag launches table
- Implemented configurable ramp steps for launch configurations
- Added at-a-glance config-managed badges on flags and features for better visibility

## Improvements

### User Interface Polish
- Consolidated otherwise/static-default/compiled-default display into single streamlined row
- Enhanced lock banner behavior to push content instead of overlaying
- Unified kill flag button styling and improved kill flag workflow animations
- Added comprehensive demo data and end-to-end tests for new override functionality
- Fixed various UI rendering issues including popover clipping and button sizing

### Command Line Interface
- Renamed `--allow-condition-overrides` flag to `--allow-runtime-overrides` for clarity
- Enhanced pb CLI with better override management capabilities
- Added `make install` target with convenient `pb` alias for easier development setup

## Bug Fixes
- Fixed nil ramp_steps causing NOT NULL constraint violations during sync operations
- Resolved issue where YAML value overrides were being dropped from bundle DefaultValue during compilation
- Fixed stale lock cleanup in development environment setup
