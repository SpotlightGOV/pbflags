# pbflags v0.8.0

This release adds a `Defaults()` null-object constructor for Go clients, introduces a release branch strategy, and improves documentation and build tooling.

## Code Generation

- **`Defaults()` constructor**: Each generated Go package now includes a `Defaults()` function that returns an implementation backed entirely by compiled defaults. This eliminates nil checks when the evaluator is optional — initialize once with `Defaults()` and call flag methods freely.

## Documentation

- **BSR setup guide**: Added example `buf.yaml` and `buf.gen.flags.yaml` configs for consumers depending on pbflags via the Buf Schema Registry.
- **`strategy: all` required**: Documented that `buf.gen` configs must use `strategy: all` so the codegen plugin can discover the `Layer` enum across files.
- **Lint tool**: Clarified that `pbflags-lint` must be run from the repository root.

## Infrastructure

- **Release branch strategy**: Minor/major releases (`vX.Y.0`) are now tagged on main and automatically create a `release/X.Y.0` branch. Patch releases are tagged on release branches. CI enforces the tag/branch pairing.
- **`make release` command**: New Makefile target for cutting releases (`make release`, `make release MAJOR=1`, `make release VERSION=vX.Y.Z`).
- **Release notes fixes**: Fixed previous-tag detection (no longer misses tags on side branches) and resolved dirty-state failure in goreleaser.
- **Dependency update**: Updated `golang.org/x/net` to v0.53.0.
