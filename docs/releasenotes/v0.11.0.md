# pbflags v0.11.0

This release introduces major architectural changes with DB-centric flag definitions, list-valued flags, a comprehensive admin web UI, and improved testing capabilities.

## Breaking Changes

- **Server architecture split**: The monolithic `pbflags-server` has been replaced with three dedicated binaries: `admin`, `evaluator`, and `sync`.
- **Deployment modes**: pbflags now operates in either standalone mode (single binary) or distributed mode (separate services).

## Migration Steps

- Update deployment configurations to use the new binary structure or choose between standalone/distributed deployment modes.
- Consult the new upgrade guide for detailed migration steps and compatibility rules.

## Server

- **New admin web UI** with flag listing, editing, and bulk override management via text paste or CSV upload.
- **List-valued flags support** allowing repeated values in flag definitions.
- **DB-centric flag definitions** enabling both monolithic and distributed deployment architectures.
- **Configurable cache TTLs** exposed as CLI flags and environment variables.
- **Improved database operations** with batched flag override updates using pipelined upserts.

## Code Generation

- **Enhanced metadata generation** with FlagDescriptors slice providing structured flag metadata per feature.
- **Improved Java codegen** with warnings for case-insensitive outer class/message name collisions.
- **Test stub generation** including Testing() function-field structs per feature for easier testing.

## Testing & Development

- **New pbflagstest package** with InMemoryEvaluator for creating test stubs.
- **Testcontainers integration** for PostgreSQL-based integration tests.
- **Playwright E2E test suite** for comprehensive admin web UI testing.
- **Pre-commit hooks** with lefthook for automated formatting and linting.

## Documentation

- **Comprehensive restructure** into dedicated reference pages with agent setup guides.
- **Updated deployment documentation** covering new monolithic and distributed modes.
- **Design documents** with standardized headers and completion status tracking.
- **Roadmap updates** based on competitive analysis and user feedback.
