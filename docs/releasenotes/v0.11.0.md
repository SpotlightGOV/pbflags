pbflags v0.11.0 introduces a new database-centric architecture with separate server binaries, adds support for repeated-valued flags, and includes a comprehensive web-based admin interface.

## Breaking Changes

- The single `pbflags-server` binary has been split into three separate binaries: `admin`, `evaluator`, and `sync`.
- Flag definitions are now stored in the database instead of being loaded from files, requiring migration of existing setups.
- Cache TTL defaults have changed from 5 minutes to 10 minutes for global state and overrides.

## Migration Steps

- Update deployment scripts to use the new separate binaries (`admin`, `evaluator`, `sync`) instead of `pbflags-server`.
- Migrate flag definitions from files to the database using the new sync workflow.
- Review the upgrade guide for detailed migration steps between deployment modes.

## Server

- Added standalone and distributed deployment modes for flexible scaling.
- Database now stores supported values for flags, eliminating file-based definition loading.
- Cache TTL values are now configurable via CLI flags and environment variables.
- Improved batch processing for flag override operations with pipelined database upserts.

## Code Generation

- Added support for repeated-valued (list) flags throughout the proto-to-codegen pipeline.
- Generated code now includes `FlagDescriptors` slice per feature for structured flag metadata access.
- Added `Testing()` function field struct per feature to simplify test stub creation.
- Java codegen now warns about case-insensitive outer class and message name collisions.

## Client Libraries

- New `pbflagstest` package provides `InMemoryEvaluator` for easy test stubbing.
- Enhanced test support with dedicated testing utilities and golden file test coverage.

## Admin Interface

- Complete web UI for listing and editing flag configurations.
- Bulk override import functionality supporting both text paste and CSV file upload.
- Integrated admin interface accessible through the new `admin` binary.

## Infrastructure

- Added Playwright end-to-end test suite for comprehensive web UI testing.
- Integrated testcontainers for PostgreSQL integration testing.
- Added pre-commit hooks with lefthook for code formatting and linting.
- Improved documentation structure with dedicated reference pages and agent setup guides.
