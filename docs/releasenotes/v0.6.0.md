# pbflags v0.6.0

This release introduces user-defined layers and a new lint tool for breaking change detection, providing more flexibility in flag configuration and better development workflows.

## Breaking Changes

- **Layer system redesigned**: The hardcoded Layer enum has been replaced with user-defined string-based layers, requiring updates to existing flag definitions and client code.

## New Features

- **pbflags-lint tool**: New CLI tool for detecting breaking changes in flag definitions with pre-commit hook integration.
- **User-defined layers**: Layers are now defined as string values instead of a fixed enum, allowing custom layer hierarchies.
- **Typed layer generation**: Code generator now creates typed layer packages from annotated enums for better type safety.

## Code Generation

- **Enhanced Go codegen**: Added layer enum discovery and validation with typed layer IDs in generated client signatures.
- **Improved Java support**: Added LayerFlag interface and typed layers with shared layerutil package.

## Server & Admin

- **Updated admin UI**: Admin interface now supports arbitrary layer names and string-based layer values.
- **Enhanced schema sync**: Sync tool updated to handle string layer values and improved update logic.
- **Layer resolution improvements**: Fixed ResolveLayer global special case handling.

## Migration Steps

1. **Update layer definitions**: Replace numeric layer references with string-based layer names in your flag definitions.
2. **Regenerate client code**: Run protoc-gen-pbflags to generate new typed layer packages and update client signatures.
3. **Update layer resolution calls**: Modify any hardcoded layer references to use the new string-based system.
4. **Optional lint integration**: Add pbflags-lint to your CI/CD pipeline or pre-commit hooks for breaking change detection.

Refer to the upgrade guide and updated documentation for detailed migration instructions.
