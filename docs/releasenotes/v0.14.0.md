# pbflags v0.14.0

This release introduces flexible cache modes and improves configuration
handling.

## Breaking Changes

- The sync tool no longer supports YAML configuration files and now uses picocli @-file flag syntax instead.

## Server

- Added write-through cache mode that bypasses TTL when `flag_ttl=0` is set.
- Implemented stale-while-revalidate behavior for cache misses to improve response times.
- Enhanced cache mode documentation covering evaluation sources and kill behavior.

## Sync Tool

- Replaced YAML configuration support with picocli @-file flag syntax for better consistency.
- Streamlined configuration file handling for command-line argument processing.

## Infrastructure

- Updated lefthook dependency to v2.
- Reorganized project structure by moving research materials to top level and clarifying documentation hierarchy.

## Migration Steps

- If you're using YAML configuration files with the sync tool, convert them to use picocli @-file syntax instead.
- Review your cache configuration if you're using custom `flag_ttl` values to take advantage of the new write-through mode.
