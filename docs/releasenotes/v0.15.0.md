# pbflags v0.15.0

This release introduces a new evaluation context system for conditional feature flags and improves the code generation experience.

## Breaking Changes

- Feature constructors now take `pbflags.Evaluator` instead of the previous interface
- Removed `Status` field from generated feature types
- Evaluation context layers have been removed in favor of a simplified context API

## New Features

- **Evaluation Context System**: Added support for conditional feature flags with evaluation context proto extensions and `Dimension` interface
- **Client Evaluator**: New `Evaluator` interface that wraps `FlagEvaluatorServiceClient` for simplified flag evaluation
- **Context Utilities**: Added `contextutil` package for managing evaluation contexts
- **Dimensions Package**: Auto-generated `dims` package from `EvaluationContext` proto messages

## Code Generation

- Constructor methods now use Go-idiomatic initialisms (ID, URL, HTTP, etc.) in generated code
- Fixed typed-nil panics in generated code
- Suppressed spurious flag type mismatch warnings on default evaluation paths

## Server

- Added wire protocol support for evaluation context fields
- Improved error handling with flag ID wrapping for evaluator errors
- Fixed `Dimension.Apply` kind-mismatch panics with better enum handling

## Infrastructure

- Enhanced end-to-end and integration tests with per-test feature isolation via `CreateTestFeature`
- Added bounded-output caching principles for launches and experiments
- Updated tooling to use lefthook v2 and improved buf formatting for multiple proto files

## Migration Steps

- Update feature constructor calls to pass `pbflags.Evaluator` instead of the previous evaluator interface
- Remove any code that depends on the `Status` field from generated feature types
- Review evaluation context usage as the layers API has been simplified
